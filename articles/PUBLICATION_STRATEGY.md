# 📚 Article Publication Strategy

To maximize engagement and ensure consistency, BRDD articles should be published following a logical dependency order.

## 🚀 Recommended Order

1.  **[BRDD: The Journey (Story)](./PTBR/BRDD-STORY.md)**
    *   **Reason:** It's the "Why". It has the least technical dependencies and sets the emotional stage.
2.  **[The Manifesto](./PTBR/BRDD-MANIFESTO.md)**
    *   **Reason:** It's the "What". It formalizes the concepts introduced in the Story.
3.  **[BRDD in Practice (The Lab)](./PTBR/BRDD-PRACTICAL-EXAMPLE.md)**
    *   **Reason:** It's the "How". It depends on the terminology defined in the Manifesto.

---

## 🔗 Link Management (Public Hosting)

When moving from local Markdown to public platforms (Medium, Dev.to, LinkedIn):

*   **Canonical Links:** Always point back to the [GitHub Hub](https://github.com/brdd-design/brdd) versions.
*   **Asset Management:** Use the URLs from the `assets/` folder in the GitHub repository (e.g., `https://raw.githubusercontent.com/brdd-design/brdd/main/assets/generic_cover_landscape.png`).
*   **Circular Dependencies:** If Article A needs to link to Article B (which isn't published yet), use a placeholder: `[Read Article B (Coming Soon)](https://brdd.design#articles)`.
